# Briefly-frontend

Next.js фронтенд для **Briefly** (демо-скелет): проекты → задачи (list/kanban/gantt), роли (бренд/сотрудник/креатор), модалка задачи с ТЗ/сценарием/чатом-заглушкой и **прикреплением файлов** (в демо сохраняются только названия).

## Стек

- **Next.js (App Router) + TypeScript**
- **shadcn/ui** (компоненты в `src/shared/ui`)
- **TailwindCSS**
- Архитектура: **FSD** (`src/shared`, `src/entities`, `src/features`, `src/widgets`, `src/app`)

## Запуск (Git Bash)

```bash
npm i
npm run dev
```

Откройте `http://localhost:3000`.

## Навигация по демо

- `/` — лендинг с быстрыми ссылками
- `/auth/sign-up` — регистрация бренда (заглушка)
- `/dashboard` — доска
- `/projects` — проекты + карточка проекта
- `/tasks` — задачи (list/kanban/gantt через `?view=`), карточка задачи в модалке
- `/creators` — база креаторов
- `/analytics` — аналитика (заглушка)
- `/settings/task-config` — типы/продукты/теги (заглушка)

## Роли (в демо)

В шапке есть переключатель роли (сохраняется в `localStorage`):

- **Креатор**: может “взять задачу” и довести до статуса “На проверке”
- **Менеджер/Админ**: принимает (“Выполнена”), может вернуть в работу
- **Аналитик**: только просмотр

## Важно про терминал в Cursor на Windows

Если у вас включена жёсткая политика выполнения PowerShell (ExecutionPolicy), команды из IDE могут падать. Для установки/запуска используйте **Git Bash** (или ослабьте политику для текущего пользователя).
